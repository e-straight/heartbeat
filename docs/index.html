Get started

Four steps to automated project updates

Install in under five minutes. No code changes, no app installations, no third-party services.

1

Install the workflow

Use the gh-aw extension to add the workflow to your repository.

gh aw add-wizard e-straight/heartbeat/heartbeat
2

Create your token

You need a personal access token so the workflow can post to your project board. Click the button below. It opens GitHub's token page with the correct scopes (project and read:org) already checked for you.

Create token (scopes pre-selected)
3

Configure your project

Set the required secrets in your repository so the workflow can post status updates.

# Set your project board URL
gh aw secrets set GH_AW_TARGET_PROJECT \
  --value "https://github.com/users/YOUR_USER/projects/YOUR_PROJECT_NUMBER"
# Set the token you created in Step 2
gh aw secrets set GH_AW_PROJECT_GITHUB_TOKEN --value "ghp_your_token_here"
4

Commit & push

Commit the generated workflow file and push to your default branch. GitHub Actions takes it from there.

git add .github/workflows/
git commit -m "Add heartbeat workflow"
git push

What it does

Automatic status updates, zero manual effort

Every meaningful event in your repo triggers an AI-generated status update on your GitHub Project board.

PR Merged

When a pull request merges, the workflow summarizes the change, author, reviewers, and target branch into a concise project update.

Issue Closed

Closed issues trigger a status post with the resolution, who closed it, assignees, and a summary of the discussion thread.

Comments Posted

New comments on issues or PRs are captured and posted as status updates, keeping your project board in sync with every conversation.

Before you begin

Prerequisites

Make sure you have these tools installed before running the setup commands.

  • GitHub CLI (gh), version 2.0 or later
  • A GitHub Project (v2) linked to your repository
  • The gh-aw extension (install below)
gh extension install github/gh-aw

Why agentic workflows

AI agents with guardrails, not blank checks

The gh-aw framework treats AI agents as untrusted by default and enforces security at every layer.

Least-privilege permissions

Agents run with read-only permissions by default. Write operations are never granted directly. They flow through safe-output jobs with scoped, validated access.

Safe outputs

Agent output is captured as structured data, scanned for threats like secret leaks and injection attacks, then executed by separate jobs, not by the agent itself.

Reproducible & auditable

Markdown workflows compile to pinned .lock.yml files with SHA-locked actions, validated schemas, and hardened configs. What you review is what runs.
